## 概要

devenv は Nix ベースの宣言的・再現可能・コンポーザブルな開発環境管理ツール。
asdf が「ツールバージョン管理」に特化しているのに対し、devenv は以下を統合的に提供する:

- ツールバージョン管理 (asdf の代替)
- 環境変数管理 (direnv 統合)
- Pre-commit フック (git-hooks.nix 統合)
- プロセス管理 (process-compose 等)
- サービス管理 (PostgreSQL, Redis 等 40+ の事前定義サービス)

## インストール方法

### 1. Nix のインストール (前提条件)

```bash
# macOS (推奨: Determinate Systems installer)
curl -sSfL https://artifacts.nixos.org/nix-installer | sh -s -- install

# Linux
sh <(curl -L https://nixos.org/nix/install) --daemon
```

### 2. devenv のインストール

```bash
nix-env --install --attr devenv -f https://github.com/NixOS/nixpkgs/tarball/nixpkgs-unstable
```

### 3. プロジェクト初期化

```bash
devenv init
```

生成されるファイル:
- `.envrc` - direnv 統合用
- `devenv.nix` - メイン設定ファイル
- `devenv.yaml` - 入力 (inputs) 設定
- `.gitignore` - 自動生成ファイル除外

## 主要ファイルの役割

### devenv.yaml

外部入力 (nixpkgs チャネル等) を定義:

```yaml
# デフォルト (明示的に指定しない場合)
inputs:
  nixpkgs:
    url: github:cachix/devenv-nixpkgs/rolling
  git-hooks:
    url: github:cachix/git-hooks.nix

# カスタム例: 安定版の nixpkgs を追加
inputs:
  nixpkgs-stable:
    url: github:NixOS/nixpkgs/nixos-24.11

# パッケージ設定
nixpkgs:
  allowUnfree: false
  allowBroken: false

# モノレポインポート
imports:
  - /shared        # git root からの絶対パス
  - ./sub-project  # 相対パス

# クリーン環境
clean:
  enabled: false
  keep: []
```

**主要オプション:**
- `inputs.<name>.url` - 入力のURL指定
- `inputs.<name>.follows` - 他の入力を参照
- `inputs.<name>.overlays` - オーバーレイ指定
- `nixpkgs.allowUnfree` - 非フリーパッケージ許可 (default: false)
- `nixpkgs.permittedUnfreePackages` - 個別許可リスト
- `imports` - ローカルインポートリスト
- `clean.enabled` - 環境クリーニング (default: false)

### devenv.nix

メイン設定。言語・パッケージ・フック・プロセス等を宣言的に定義:

```nix
{ pkgs, config, ... }:

{
  # パッケージ
  packages = [
    pkgs.git
    pkgs.jq
    pkgs.docker-compose
  ];

  # 環境変数
  env = {
    PROJECT_NAME = "my-project";
    NODE_ENV = "development";
  };

  # カスタムスクリプト
  scripts = {
    dev.exec = "npm run dev";
    test.exec = "npm test";
  };

  # シェル起動時の処理
  enterShell = ''
    echo "Development environment activated"
  '';

  # 言語設定
  languages.javascript = {
    enable = true;
    package = pkgs.nodejs_22;
    bun = {
      enable = true;
      install.enable = true;  # 自動で bun install 実行
    };
  };

  languages.python = {
    enable = true;
    version = "3.13";  # nixpkgs-python で自動解決
    uv = {
      enable = true;
      sync.enable = true;  # 自動で uv sync 実行
    };
  };

  languages.deno = {
    enable = true;
  };

  languages.typescript = {
    enable = true;
  };

  # Git フック
  git-hooks.hooks = {
    # ... (後述)
  };

  # プロセス管理
  processes = {
    # ... (後述)
  };
}
```

### devenv.lock

`devenv.yaml` の入力を特定のコミットリビジョンに固定。
`devenv update` で更新。Git にコミットして再現性を保証。

### devenv.local.nix / devenv.local.yaml

個人用のローカルオーバーライド。自動的に `.gitignore` される。

## 言語設定の詳細

### Node.js / JavaScript / Bun

```nix
languages.javascript = {
  enable = true;

  # Node.js パッケージ (デフォルト: pkgs.nodejs-slim)
  package = pkgs.nodejs_22;  # Node.js 22.x

  # Bun
  bun = {
    enable = true;
    package = pkgs.bun;      # カスタムパッケージ指定可能
    install.enable = true;    # bun.lockb 変更時に自動 install
  };

  # npm (必要な場合)
  npm = {
    enable = false;
    install.enable = false;
  };

  # Corepack (pnpm/yarn のバージョン管理)
  corepack.enable = false;

  # プロジェクトディレクトリ (モノレポ用)
  directory = "";  # デフォルト: プロジェクトルート

  # LSP
  lsp.enable = true;
  lsp.package = pkgs.typescript-language-server;
};
```

**注意**: Node.js には `languages.python.version` のような直接的なバージョン指定オプションがない。
`package = pkgs.nodejs_22;` のように nixpkgs のパッケージ名で指定する。
利用可能な Node.js パッケージ: `nodejs_18`, `nodejs_20`, `nodejs_22` 等。

### Python

```nix
languages.python = {
  enable = true;

  # バージョン指定 (推奨: nixpkgs-python で自動解決)
  version = "3.13";         # "3.13" または "3.13.9" 等

  # または直接パッケージ指定
  # package = pkgs.python313;

  # uv パッケージマネージャー
  uv = {
    enable = true;
    package = pkgs.uv;
    sync = {
      enable = true;        # 自動 uv sync
      allExtras = false;
      allGroups = false;
      extras = [];
      groups = [];
      packages = [];        # ワークスペースパッケージ
      arguments = [];       # 追加 CLI 引数
    };
  };

  # Poetry (代替)
  poetry = {
    enable = false;
    activate.enable = false;
    install.enable = false;
  };

  # venv
  venv = {
    enable = false;
    quiet = false;
    requirements = null;    # pip requirements
  };

  # ネイティブライブラリ
  libraries = [];

  # LSP
  lsp.enable = true;
  lsp.package = pkgs.pyright;

  # manylinux 互換
  manylinux.enable = false;
};
```

### Deno

```nix
languages.deno = {
  enable = true;
  package = pkgs.deno;  # カスタムパッケージ指定可能
};
```

**注意**: Deno にはバージョン指定オプションがない。
特定バージョンが必要な場合は nixpkgs の別リビジョンからパッケージを取得する必要がある。

### TypeScript

```nix
languages.typescript = {
  enable = true;
  lsp.enable = true;
  lsp.package = pkgs.typescript-language-server;
};
```

## バージョン固定の方法

### 方法 1: languages.*.version (Python のみ)

```nix
languages.python.version = "3.13.9";
```

Python のみ `nixpkgs-python` による自動バージョン解決をサポート。

### 方法 2: nixpkgs のパッケージ名で指定

```nix
languages.javascript.package = pkgs.nodejs_22;
languages.deno.package = pkgs.deno;
languages.python.uv.package = pkgs.uv;
languages.javascript.bun.package = pkgs.bun;
```

利用可能なパッケージは `devenv search <name>` で検索可能。

### 方法 3: 別の nixpkgs リビジョンから取得

特定バージョンが必要な場合:

```yaml
# devenv.yaml
inputs:
  nixpkgs-pinned:
    url: github:NixOS/nixpkgs/SPECIFIC_COMMIT_HASH
```

```nix
# devenv.nix
{ inputs, pkgs, ... }:
let
  pkgs-pinned = import inputs.nixpkgs-pinned { system = pkgs.stdenv.system; };
in {
  languages.deno.package = pkgs-pinned.deno;
}
```

### 方法 4: オーバーレイ

```nix
# devenv.nix
{
  overlays = [
    (final: prev: {
      nodejs = prev.nodejs_22;
    })
  ];
}
```

## direnv 統合 (.envrc)

### セットアップ

1. direnv をインストール
2. シェルフックを追加 (`.bashrc` / `.zshrc`)
3. `.envrc` を作成

```bash
# .envrc (devenv v1.4+)
#!/usr/bin/env bash
eval "$(devenv direnvrc)"
use devenv
```

4. `direnv allow` で有効化

### 動作

- ディレクトリに入ると自動的に開発環境がアクティベート
- 離れると自動的にディアクティベート
- `.direnv/` を `.gitignore` に追加 (devenv init で自動追加)

## Pre-commit フック (git-hooks)

devenv は [git-hooks.nix](https://github.com/cachix/git-hooks.nix) を統合。
v1.11 からデフォルトで `prek` (Rust 実装) を使用。

```nix
git-hooks.hooks = {
  # === このプロジェクトに関連するフック ===

  # JavaScript/TypeScript
  biome.enable = true;        # Biome (lint + format)
  # eslint.enable = true;     # ESLint (代替)
  # prettier.enable = true;   # Prettier (代替)

  # Python
  ruff.enable = true;         # Ruff (lint + format)
  # black.enable = true;      # Black (format, 代替)
  mypy.enable = true;         # MyPy (型チェック)

  # Shell
  shellcheck.enable = true;

  # Nix
  nixpkgs-fmt.enable = true;

  # カスタムフック
  custom-lint = {
    enable = true;
    name = "Custom Lint";
    entry = "make lint";
    files = "\\.(ts|tsx|py)$";
    language = "system";
    pass_filenames = false;
  };
};

# prek (デフォルト, Rust実装で高速) を pre-commit に戻す場合:
# git-hooks.package = pkgs.pre-commit;
```

**動作:**
- `devenv shell` 実行時に `.git/hooks/pre-commit` に自動インストール
- `devenv test` で CI でも実行可能
- `.pre-commit-config.yaml` は自動生成 (Git 管理不要)

## プロセス管理

`devenv up` で定義したプロセスを起動:

```nix
processes = {
  backend = {
    exec = "docker-compose -f ./docker-compose.backend.yaml up";
    # working_directory = config.git.root;  # git root を参照
  };

  frontend = {
    exec = "cd frontend && bun run dev";
  };

  storybook = {
    exec = "cd frontend && bun run storybook";
  };
};
```

### プロセス設定オプション

```nix
processes.myservice = {
  exec = "command here";

  # リスタートポリシー
  # "on_failure" (default) | "always" | "never"
  restart = "on_failure";
  max_restarts = 5;      # リスタート回数制限

  # 依存関係
  after = [ "devenv:processes:database" ];  # database 起動後に開始

  # ファイル監視 (変更時に自動リスタート)
  watch = {
    paths = [ ./src ];
    extensions = [ "ts" "tsx" ];
    ignore = [ "node_modules" "*.log" ];
  };
};
```

### 事前定義サービス

```nix
# PostgreSQL (例)
services.postgres = {
  enable = true;
  initialDatabases = [{ name = "myapp"; }];
};

# Redis
services.redis.enable = true;

# 他: MySQL, MongoDB, Elasticsearch, RabbitMQ 等 40+ サービス
```

### プロセスマネージャー選択

```nix
process.manager.implementation = "process-compose";
# 選択肢: process-compose, overmind, honcho, hivemind, mprocs
```

## asdf からの移行時の注意点

### 利点

| 項目 | asdf | devenv |
|------|------|--------|
| **再現性** | .tool-versions | devenv.lock (Nix ハッシュベース) |
| **環境変数** | 別途 direnv 設定必要 | 統合済み (env オプション) |
| **Pre-commit** | 別途設定必要 | 統合済み (git-hooks) |
| **プロセス管理** | なし | 統合済み (devenv up) |
| **サービス** | なし | PostgreSQL 等 40+ 事前定義 |
| **起動速度** | プラグイン依存 | 100ms以下 (キャッシュ有効時) |
| **モノレポ** | 限定的 | imports で構成管理 |
| **CI/CD** | 要プラグイン | devenv test で統合 |

### 制約・懸念点

1. **Nix の学習コスト**: Nix 言語の基本理解が必要
2. **マイナーバージョン固定の困難さ**: Node.js は `nodejs_22` (22.x系) の粒度で、`22.21.1` のような厳密な固定は nixpkgs のコミット固定が必要
3. **Python バージョン**: `languages.python.version = "3.13.9"` は nixpkgs-python で対応可能だが、完全な互換性は要確認
4. **Deno バージョン固定**: 直接的なバージョン指定なし。nixpkgs のコミット固定で対応
5. **uv バージョン**: `languages.python.uv.package` で指定するが、nixpkgs の提供バージョンに依存
6. **ディスク使用量**: Nix store は大量のディスクを消費する可能性あり
7. **チームメンバー全員に Nix インストールが必要**

### 移行手順 (提案)

1. Nix + devenv をインストール
2. `devenv init` でファイル生成
3. `devenv.nix` に言語・パッケージ設定を記述
4. `devenv shell` で動作確認
5. direnv 設定 (.envrc) で自動アクティベーション
6. `.tool-versions` と asdf プラグイン設定を削除
7. Makefile の `init` ターゲットを devenv 対応に更新
8. チームメンバーへの Nix + devenv インストール案内
