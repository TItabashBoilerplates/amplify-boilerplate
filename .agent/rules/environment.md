# Environment Configuration

Secrets and non-secret configuration are managed differently in the Amplify stack.

## Secrets — Amplify secrets (SSM Parameter Store)

Secrets (API keys, tokens, third-party credentials) are stored in **Amplify secrets**, backed by
AWS SSM Parameter Store. There are no `.env.secrets` files and no external secret manager.

```bash
# Set a secret for the per-dev cloud sandbox
ampx sandbox secret set NAME
```

In the Amplify backend definition (`frontend/packages/backend/amplify/`), reference secrets with the
`secret()` helper:

```typescript
import { secret } from '@aws-amplify/backend'

// e.g. inside a function/resource definition
environment: {
  SOME_API_KEY: secret('SOME_API_KEY'),
}
```

## Non-secret Configuration

- **Frontend public config**: standard Next.js / Expo environment variables (e.g. `NEXT_PUBLIC_*`).
- **Amplify outputs**: `amplify_outputs.json` is generated by `ampx sandbox` (or pipeline deploy) and
  wires the frontend to Cognito / AppSync / S3. It is regenerated automatically — do not edit by hand.

## Notes

- ⚠️ Running `ampx sandbox` / `ampx sandbox secret set` requires valid AWS credentials (a profile).
- Branch/production deploys resolve secrets from the matching SSM Parameter Store entries for that
  Amplify app/branch.
